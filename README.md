### Задание 2. Транзакции

2.1. Пользователь пополняет баланс счёта телефона, распишите пошагово, какие действия должны произойти для того, чтобы 
транзакция завершилась успешно. Ориентируйтесь на шесть действий.

2.1.* Какие действия должны произойти, если пополнение счёта телефона происходило бы через автоплатёж?

*Приведите ответ в свободной форме.*

### Решение:

2.1

1) Инициация операции
Пользователь:
- вводит номер телефона,
- указывает сумму пополнения,
- выбирает способ оплаты (банковская карта, СБП и т.д.),
- подтверждает намерение оплатить.

2) Проверка корректности данных
Платёжная система выполняет:
- валидацию номера телефона,
- проверку формата суммы,
- проверку лимитов,
- антифрод-проверку.

3) Авторизация платежа банком
Банк
- проверяет наличие средств,
- проверяет статус карты,
- резервирует сумму.

4) Подтверждение и списание средств
- банк списывает средства,
- платёжная система фиксирует успешную транзакцию,
- формируется платёжный идентификатор (transaction ID).

5) Подтверждение и списание средств
Оператор:
- принимает платёж,
- увеличивает баланс абонента на указанную сумму,
- фиксирует операцию в своей биллинговой системе.

6) Завершение транзакции и уведомление клиента
Система:
- получает подтверждение от оператора,
- фиксирует статус «Успешно»,
- отправляет пользователю уведомление (SMS / push / email),
- формирует чек (в соответствии с требованиями законодательства).

### Задание 4. Кластеры

Необходимо производить большое количество вычислений при работе с огромным количеством данных, под эту задачу 
выделено 1000 машин. 

На основе какого критерия будете выбирать тип СУБД и какая модель *распределённых вычислений* 
здесь справится лучше всего и почему?

*Приведите ответ в свободной форме.*

# Решение:

Главный критерий выбора — горизонтальная масштабируемость, так как необходимо эффективно распределять данные между узлами, выполнять параллельные вычисления, минимизировать сетевые издержки и обеспечивать отказоустойчивость системы.

В данном случае оптимально подойдёт NoSQL column-oriented СУБД, поскольку она эффективно обрабатывает большие объёмы данных, считывая только необходимые столбцы, что значительно ускоряет массовые аналитические вычисления и хорошо сочетается с распределённой моделью обработки данных
